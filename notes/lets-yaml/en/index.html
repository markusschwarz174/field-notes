YAML for teaching

Four days before school starts, an uploaded pseudo-YAML becomes a teaching construction kit: for tasks, language-sensitive teaching, interdisciplinary learning checks and a colleague.

01

When You Have an Idea

02

Do You Always Have to Get Frustrated First?

It's mid-August, and more times than I can count over the past few days I've wanted to slam the laptop shut. The Geo-Chat has grown so long that I can't find my worksheet prompts in it anymore. And when I do find one and reuse it as a 1:1 copy, the worksheet with that exact same prompt looks different from the one before.

The reason the chats got so long is entirely on me. At the start of August I began prepping lessons for Geography, topic: "Our Earth." All of it done with GPT in the browser. It was back-and-forth ping-pong, and that went on until I finally got the output I wanted.

I want a worksheet – ping-pong – I have the worksheet.
I have a question – ping-pong – eventually I get an answer I actually understand.

It's dead simple — but this way of working has a few downsides:
1. The GPT chats keep getting longer and longer.
(=> The Geo-Chat alone ran to 105 Word pages when exported.)
2. Where's my "worksheet prompt"? And the searching begins.
3. The prompts get longer and more detailed too — but not better.
4. Thematically, the chats are a historically grown ratatouille of all kinds of topics.
And at some point, nothing works anymore.

With the Geo-worksheet I see it clearly: same input, but the layout is different, an element is missing, something never quite lines up. It's so frustrating, because I think in terms of "input = output" — but that's apparently not how this works. It still adds things, interprets things differently than I do. That sometimes drives me to despair.

I'm stuck in the tension between AI's generativity and my wish to treat GPT like a program I can program. But I'm not looking for a solution — I'm just venting. It looks so simple, and yet I can't get it to do what I want.
GPT is a great tool. That it could also be a tool for solving problems — that I haven't figured out yet.

Maybe it was coincidence, maybe the time was just right — last night, while I was going through the list of task formats for a geography worksheet, the motion sensors in my wardrobe suddenly popped into my head. Associations you don't understand and don't need to understand.
And it doesn't matter, because now I have an idea: away from writing prompts, toward writing code. I'm no longer thinking "How can I create a worksheet?" — I'm thinking "How could I write rules that then create a worksheet on their own?"

03

My Very Own EUREKA Moment

It's Thursday morning, August 14, 2025, and the clock says 8:25 a.m. Experiment begins. I start a new chat and open Word at the same time. This time I'm not typing the prompt directly into the browser prompt window — I'm drafting it in Word.
I want to try out the YAML code idea and just start writing: "task_formats:" — new line — "task_type:" — new line — and so on.

I write as if I'm filling out index cards at a library. After three task formats, I check the screen. First: it looks nothing like the Home Assistant interface. Second: what I've written looks ugly. The code is almost certainly wrong and I'd need to fix it. But I can't fix it, because I don't know how. I can tell the spacing and the brackets are off — but I don't know how many spaces or which brackets would be right. Typical half-knowledge. Doesn't matter.

I upload the Word document to GPT without any comment and press — it's 9:05 a.m. — ENTER.

I sit there and watch GPT write line by line, and I don't understand why it's doing that — because I didn't type anything into the prompt window.
"Analyzing Word document" or "The code in the Word file contains errors." That's what I expect from GPT. But that response never comes. The fact that it not only recognizes the broken lines from the Word file as YAML and corrects them on its own, but also interprets the content and generates a worksheet from it — that leaves me speechless.
It takes me a moment to realize it actually worked.
GPT has created a geography worksheet. I'm still staring at the screen like I've turned to stone, thinking: "I programmed GPT!"
I ask why it was able to read code written in YAML, since I'd assumed I had to write in "plain language." GPT explains that it can interpret and process other formats too — YAML, Markdown, Python, JSON — as structural cues. I have to rethink everything. I'm no longer thinking: "I need a worksheet." Or: "I want a test." Instead, I build a template, upload it — and GPT produces a complete worksheet in one step, following my rules, repeatably. Or I take an existing library and use it in a similar but different structure. I'm done with ping-pong.

04

From Hack to System

05

My Rules Become Libraries

That Thursday, August 14, 2025, changes my prompting and the way I communicate with GPT. I struggle through two more rounds, writing in pseudo-YAML. Then it saves me by accident. I apologize for all the syntax errors in the YAML code. GPT says: "No problem, I can write that for you." I have to laugh — I never would have thought of that.

From that point on, I use GPT as my personal code writer. YAML is just a data format, not a programming language. But it still feels like programming to me.
The rules as the smallest unit, the library as the collection, and the generator as the workflow that builds worksheets from all of it. I want to create libraries for everything in my classroom that I can put into a structure. And I already have a clear picture of what I want to represent as YAML in my teaching.

First, I build a base structure for the assignment dossiers:

Grundstruktur:
schema_version: 1
dokument_typ: "Arbeitsdossier"
fach: "Raum-Zeit-Gesellschaft"
niveau: "B"
klasse_alter: 13
ausgabeformat: "Word-ready"
quellen:
  - name: "Revolution der Erde"
    seiten: "S. 12–13"
  - name: "Beleuchtung und Erwaermung der Erde"
    seiten: "S. 22–23"

Then I define the format rules for the tasks.

formatregeln:
  punkteanzeigen: false
  platz_fuer_antworten: true
  starticons: { einfach: "🟡", mittel: "🟢", schwer: "🟣" }
  nummerierung: "A, B, C, D, (E)"
  ueberschrift_schema: "Lernziel {n}"
  schwierigkeit:
    A: [I, II]       # einfach
    B: [II]          # einfach
    C: [III, IV]     # mittel
    D: [III, IV]     # mittel
    E: [V, VI]       # schwer (Transfer)
  quellenregel: "keine Mischquellen innerhalb eines Lernziels"
  offene_vs_geschlossene:
    einfach_mittel: "geschlossen bevorzugt"
    schwer: "offen"

Then a block for the task formats (excerpt):

aufgabenformate:
  - code: markieren
    icon: "🔠"
    beschr: "Markieren/Unterstreichen"
  - code: lueckentext
    icon: "📝"
    beschr: "Lueckentext mit Wortpool"
  # ... weitere Formate

And for the learning objectives:

lernziele:  # Nomen + Verb, 10-15 Woerter, Bloom-gerecht
  - text: "Wortarten in Saetzen bestimmen und korrekt benennen."
    kompetenz: "Sprachgebrauch"
    quelle: "Revolution der Erde"
  - text: "Satzbau erkennen und einfache Saetze sicher umformen."
    kompetenz: "Grammatik"
    quelle: "Revolution der Erde"

I spend the rest of the day generating lines of code like this. These are my "library cards." All I change are the pages and the learning objectives. Then I write in the prompt: "Generate the test based on the YAML above." Here's the learning objectives block. And GPT produces a test for me.

Mid-afternoon, something occurs to me. I think back to last week: "Remember how you created those geo worksheets and they all looked the same, because GPT kept using the same task types in the same order every single time?"

I come up with an idea and have it build a rotation matrix:

rotationsmatrix:
  A: [markieren, lueckentext, fehlerjagd, zuordnung, diagramm]
  B: [zuordnung, mehrfach, lueckentext, markieren, regel]

It works. The afternoon is only interrupted by a phone call from my boss about a form that wasn't filled out correctly. Three calls in total — they briefly pull me away from the work, and I find myself wondering whether I'm at the right employer. By late afternoon, my focus, my full concentration, is back on the master prompt "Create learning dossier." It's an experiment. What input gets GPT to produce what output?

No more ping-pong. Now I'm thinking through workflows and testing them. And every time, the small libraries — learning objectives, task formats, rotation matrix — are right there alongside, and they keep growing. With each chat they grow not just in content, but in number.

By evening I've essentially built a worksheet generator, and I start asking myself: "Why are you doing this?" "There are thousands of tools like this online." "This is a waste of time." "The commercial generators are faster, offer more, and have a nicer frontend." "Your work is pointless."

I think about it for a moment. "I'm going to keep going."

Because there's one thing the commercial tools can't do: show me which rule changes what. It's the autonomy, the transparency, and the adaptability that set my code apart from the rest. From math to geography — from learning objectives to task formats.
I want to see, to control, to understand —
and above all, to decide for myself.
That's my most important job as a human being.
I don't want to use a black box. So I keep building — with GPT, but by my own rules. I'm the pilot. I'm in the cockpit, not the passenger seat.

It's gotten late again. I shut down the computer, settle into the armchair in the sunroom, and look out at the mountains. The view is the same, but the feeling inside is different. My eyes say: "The mountains are right where they've always been." And a voice inside says: "You moved them — just a little, but they moved."

06

Language in Mathematics

I'm up early the next morning. It's Friday, August 15. I still have today, Saturday, and Sunday before school starts again. Three weeks ago I made a work plan, and today was marked "RESERVE." I wasn't supposed to need to work today at all. In theory. But yesterday changed the plan. What was zero hours is now 24+. Back in July I assumed AI would reduce my workload and cut my working hours. I've noticed nothing of the sort — and it doesn't bother me in the least, because something new is more important:
For many years I've had the privilege of working with a wonderful special education teacher — let's call him Reed. We often meet during breaks behind the school building and talk about pedagogy or teaching methods. I remember clearly how Reed brought up the importance of conceptual understanding in math class more than once. He'd seen how many students struggle with it.
It was yesterday evening in the sunroom when that conversation with Reed came back to me, and I said to myself: "You've got a code for building geography worksheets, and you've got the planning materials for the next math unit. What happens if you use the geography YAML not to build a geography worksheet, but a worksheet about math vocabulary? Basically a language worksheet for math."
On this Friday morning
I can't help myself —
I have to go down that road.
So I spend more or less the entire day reading up on language-sensitive teaching and working with GPT to develop exactly that kind of worksheet. That day I read more than I worked — but by evening the worksheet was ready to print. I can't say that about many projects.

I took the old geography worksheet and rebuilt it as a language worksheet for math — German-language exercises, but with math content. Nothing fancy. A worksheet with one specific job. Here's what it looked like:

I thought the worksheet was great and used it the following week — but the students weren't impressed. Their feedback: too much writing.
I accept that. And I understood why: by the end there was just too much paper — worksheets, dossiers, learning-goal cards, and so on.
The idea is good and sound, but it can't be designed as an extra task piled on top. I think language support could work if it's built into the subject task rather than placed alongside it as something separate. It needs to be embedded. I need to figure out how to integrate that into the next work dossier.
So in the end it was a great sheet —
but one sheet too many.

07

The Interdisciplinary Quiz

It's Friday afternoon. I'm still working on the language-sensitive math sheet when another idea hits: an interdisciplinary quiz.
Why always practice spelling or grammar in a German-class context? Why not build a quiz on parts of speech using content from the geography unit "Our Earth"?
Parts of speech aren't practiced here as an isolated grammar island — they're practiced on sentences that come up in subject lessons anyway. I adjust the schema and build the first version: "Interdisciplinary German Test." Horrible name. I'll change it.

schema_version: 1
klasse: "7–8"
niveau: "B"
thema: "Erde & Sonne – Grundlagen"
lernziele:
  - "Wortarten in kurzen Fachsätzen sicher benennen."
  - "Satzglieder erkennen und markieren."
seitenplan:
  - titel: "Einführung"
  - titel: "Wortarten – Grundlagen"
  - titel: "Satzglieder – Anwendung"
layout:
  zeilenabstand: 1.15
  schrift: "Calibri 11"
icons:
  einfach: "🟡"
  mittel: "🟢"
  schwer: "🟣"

I look at the test on screen and ask myself: "Is this a geography test about spelling, or a spelling test about geography?"

I'll put together four versions between August and December, and expand the idea to writing assignments as well. For example, using the history topic "Explorers and Conquerors": a "Harbor Dialogue," an "Explorer's Travel Journal," a "Newspaper Report," "Navigation — Then and Now." It works really well. I've had the chance to read the students' texts.

08

A Prompt Other People Can Use

I have a wonderful colleague — let's call him Steve — who also teaches math and German at my school. I want to share the language-math dossier idea with him. "Does my library work when someone else uses it?"
So the question isn't: "How do I use the code to build a worksheet?" The question is: "Can someone else build a worksheet with my system without knowing the whole backstory?"
There's one small but significant obstacle: I can't just send him the master prompt. It's a working tool, not a communication instrument. It's a PDF I upload.

I know the architecture of the whole prompt. Steve only knows which file to upload. When GPT sees that file, key information is missing: Which class? Which learning objectives? Which language hurdles? The prompt itself has to handle exactly those questions — by having the AI take over the conversation with the user. I need a defined flow for the "user with GPT" dialogue, and I have to map that interaction inside the prompt.

Steve should only have to upload the PDF and type "Read the prompt." From there, GPT presents a personalized welcome message, asks the missing questions, and uses the answers to generate the materials.

On this Sunday evening, I still have no idea what these small libraries will eventually become. I also don't know yet that in November I'll build a small assignment sheet — this time for a learning workshop. And from that assignment sheet, a learning space will grow, and then a learning room.

YAML will no longer just describe tasks —
it will help map out learning paths.
