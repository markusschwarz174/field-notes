This page is a look behind the scenes of the Field Notes. The German texts come first. Then comes P7: the translation. But P7 isn't simply a button labeled "English." P7 is its own phase: preparation, first translation, cross-check, patch, and a gate. The story now has three versions: first the manual browser loop, then the shaky CLI detour, now the working SDK route.

Why I need a translation workflow in the first place

The Field Notes are written in German. Not academic. Not polished. They live on small movements: a sentence that first seems trivial and then suddenly matters; images from the classroom; my wonder, my suspicion, my impatience.

When I have it translated, I can't just ask: Is the English correct? The real question is harder:

Does the text still sound like me,
or does it only sound like a very polite AI?

That's why P7 works on a simple principle: One AI translates. Another AI reviews. The first AI patches. The tool running in the background keeps track of everything that happens: files, blocks, reports, versions.

Why "upload to GPT and let it translate" isn't enough

The obvious route is tempting: I take a finished German note, upload it to ChatGPT, type "Translate this to English," copy the result back, and publish. For an email, that would probably be enough. For the Field Notes, it isn't.

The problem isn't that GPT translates badly. The problem is that a chat window doesn't replace a process. There's no reliable data contract, no block count, no second look from a different model, no clean trail back to the German source.

The manual browser loop from June 28 still mattered. That's where the data contract took shape: PASS 0 through PASS 4, numbered blocks, separate prompts, downloads, cross-check, patch. But this route turns Markus into a file courier. It shows what P7 needs. It is not the solution for ongoing production.

PASS 0

The German source text is prepared: Markdown, blocks, metadata, material for translation.

PASS 1

Claude produces the first English translation. Not as flowing prose, but block by block.

PASS 2

GPT checks the English version against the German original and writes up a list of issues.

PASS 3

Claude gets the issue list back and corrects the affected passages specifically.

PASS 4

The reviewed blocks become the publishable English Markdown file.

The real gain isn't in the browser. It's in the principle behind it: the translation isn't just a response — it's a chain with verifiable handoffs.

Nine reasons against the chat-window route

The short version: a chat window works fine for a single text. P7 needs a production line. The longer version is nine very practical reasons.

01 Structure

The notes consist of sections, images, captions, prompt/response passages, and frontmatter. A free chat can flatten, compress, or quietly rearrange all of that.

02 Completeness

Without block numbers, there's no reliable way to catch whether a paragraph is missing, duplicated, or slipped into the wrong order.

03 Voice

The Field Notes can't sound like international AI-brochure copy. The workshop tone has to be held explicitly — and checked.

04 Counter-model

A single model doesn't review its own translation hard enough. P7 needs cross-model: Claude writes, GPT checks, Claude patches.

05 Logs

After a chat export, it's unclear which version, which prompt, and which response led to the published text.

06 Repetition

When the German source gets updated, the translation has to run again — reproducibly. Copy-paste in a browser is not a re-run.

07 Limits

Long notes push against output limits. A browser can cut off, truncate, or need a "continue" — without the process catching it cleanly.

08 Series

Across many notes and languages, you need consistent folders, consistent names, consistent reports. Otherwise every translation becomes a one-off.

09 Terms

Werkstatt, Passung, didaktische Reduktion, Field Note: terms like these need to stay consistent across the series. A chat prompt forgets that fast.

The question isn't:
Can GPT translate?
The question is:
Can I still trust the path that got me there?

That's why "upload to GPT" isn't a bad idea — it's just the wrong route for this job. It optimizes the first click. P7 has to optimize the final audit trail.

The SDK route works

The next attempt doesn't replace the P7 principle — it replaces the transport layer. Instead of browser windows and downloads, a Python script calls the model APIs directly: Anthropic for translation and patch, OpenAI for the review pass. The blocks stay blocks. Responses have to come back as JSON. Costs and run times go into metrics.

The SDK route works. A 3,000-word note can run in minutes instead of half an hour. Every pass produces files. Every run produces metrics. If a model turns 79 blocks into 82, the script can count. If GPT finds issues, they sit in a file. If Claude patches, you can see later which blocks were changed.

The three versions of the translation

Looking back, P7 isn't a single technique — it's a small development history. Each version solved something and, at the same time, exposed the next gap.

Version 1

Browser-manual: Claude translates, GPT reviews, Claude patches. The principle is right, but Markus is carrying files between windows.

Version 2

CLI/PYCLI route: the file courier was supposed to disappear. The idea is right, but the route is too fragile on Windows and with subprocesses.

Version 3

SDK route: Python calls Anthropic and OpenAI directly. Blocks, JSON, logs, and metrics become part of the process.

What's gained

The translation becomes repeatable. Not just the text matters — so does the path it took to get there.

Control

Structured Output significantly reduces the old error class of "prose instead of JSON".

Series

For EN, FR, and ES, there's now a process that doesn't start from scratch every time.

The third version is the point
where translation stops being chat work
and becomes a pipeline.

Why I still don't trust the translation blindly

The dangerous sentence would be: The AI translated it, so it must be fine. That's exactly not how P7 works.

The translation is broken into blocks. Every block has a number. That sounds dry, but it matters. An AI can be poetic, friendly, confident — and still drop a paragraph or shift a meaning. Numbered blocks turn trust into a counting task.

I don't trust the AI
because it sounds convincing.
I trust the process only as far
as it leaves a trail.

Then comes the second check: a different model. Claude translates. GPT reads German and English side by side and looks for meaning shifts, loss of tone, broken structure. In the test run, GPT found real problems. Not everything. But enough to make the process better than any single polished answer.
